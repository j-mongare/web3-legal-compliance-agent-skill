# DAO Governance Red Flags (Legal Risk)

- Small number of multisig signers (especially founder-heavy)
- No timelock on upgrades or admin functions
- Emergency pause / blacklist controlled by individuals
- Treasury withdrawal rights without governance vote
- Off-chain coordination that overrides on-chain votes
- High voting power concentration or delegation to few addresses
- Hidden admin keys or proxy admin control
- Does the DAO have a legal wrapper?
- Can founders/founding entities veto token holders?
- Can founders arbitrarily reroute revenue, depracate tokens, engage other entities without consulting token holders?
